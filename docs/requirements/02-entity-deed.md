# 02. Сущность «Дело»

## Состав дела

Дело состоит из следующих элементов:

### Эмодзи или символы

Визуальный идентификатор дела (иконка, эмодзи или короткий символ) для быстрого распознавания в списке.

### Название

Краткое название дела (например «Утренняя зарядка», «Настроение за день»).

### Описание

Текстовое описание дела: для чего форма, как её заполнять. Необязательное поле.

### Блоки вопрос-ответ

Набор блоков, каждый из которых задаёт один вопрос с определённым типом ввода. По умолчанию при создании дела добавляется один блок типа «Число».

### Действие «+ Добавить блок вопрос-ответ»

Пользователь может добавлять новые блоки вопрос-ответ. Тип каждого блока выбирается из списка: Число, Текст (строка), Текст (абзац), Один из списка, Несколько из списка, Шкала, Да/Нет.

### Удаление блоков

Блок с вопросами можно удалять при создании дела и во время редактирования дела. После удаления блок исчезает из формы. В существующих записях данные по удалённому блоку должны храниться для истории: сохраняются название блока, введённое значение и дата удаления блока.

## Связь с записями

К одному делу привязано множество записей — каждая запись есть один момент заполнения формы по этому делу.

## Влияние редактирования дела на данные

Что происходит с записями и отображением при изменении названия, удалении или добавлении блоков и при попытке изменить тип блока — см. [13-rules-editing-deed-data.md](13-rules-editing-deed-data.md).
