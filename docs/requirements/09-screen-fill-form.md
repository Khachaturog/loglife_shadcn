# 09. Экран: заполнение формы

## Назначение

Экран, на котором пользователь в нужное время заполняет форму выбранного дела: вводит или выбирает ответы по каждому блоку и сохраняет новую запись.

## Элементы экрана

- **Контекст дела** — карточка над полями формы: эмодзи (в круге), название и при наличии описание (до нескольких строк с обрезкой длинного текста; полный текст в подсказке при наведении).
- **Дата и время** — дату и время записи можно редактировать (например, если пользователь добавляет данные за вчера).
- **Блоки в порядке, заданном в деле**:
  - Каждый блок отображается согласно своему типу:
    - Число — поле ввода числа и кнопки ± (один шаг по тапу или ускоренное изменение при удержании, как в виджете «Кликер»).
    - Текст (строка) — однострочное поле.
    - Текст (абзац) — многострочное поле.
    - Один из списка — радио или выпадающий список вариантов.
    - Несколько из списка — чекбоксы вариантов.
    - Шкала — шкала с подписями первого и последнего значения; на узком экране сегменты уже по ширине (как у компактного размера темы), по высоте — как у основных контролов, чтобы удобнее нажимать; с брейкпоинта `sm` темы визуально как остальные поля формы.
    - Да/Нет — переключатель или два варианта выбора.
  - Обязательные блоки помечаются (например, звёздочка или подпись «Обязательное»).
  - У каждого блока, по которому уже есть ответ в форме, может отображаться кнопка **«Сбросить»** — она очищает значение этого блока (как при ручной очистке поля: для числа ключ ответа убирается из состояния по тем же правилам, что и при пустом вводе).
- **Блок «Заметки»** — технический блок внизу формы, свободный текст (см. [11-technical-defaults.md](11-technical-defaults.md)).
- **Кнопка «Добавить запись» — отправка формы.

## Поведение при сохранении

- Проверка обязательных полей ([06-rules-required.md](06-rules-required.md)): при попытке сохранить с пустыми обязательными блоками запись не отправляется; под каждым таким блоком показывается подсказка.
- Если в деле **нет** обязательных блоков, для сохранения нужен хотя бы один непустой ответ; при попытке отправить пустую форму показывается соответствующее сообщение.
- Кнопка отправки доступна до успешной отправки (кроме состояния «идёт сохранение»), чтобы по нажатию отобразилась валидация.
- При успешном сохранении: запись создаётся с текущими датой и временем, пользователь перенаправляется (например, на историю записей или список дел) или видит сообщение об успехе.
- На устройствах с поддержкой [Vibration API](https://developer.mozilla.org/en-US/docs/Web/API/Vibration_API) (часто Android Chrome в **защищённом контексте**: HTTPS или localhost): короткая тактильная отдача при выборе значений в контролах формы, при первом нажатии на ± числа и при успешном сохранении. В **iOS Safari** веб-вибрации нет (API не реализован); звуковой fallback не используется — только вибрация там, где API доступен.
