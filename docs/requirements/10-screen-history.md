# 10. Экран: история записей

## Назначение

Блок «история записей»: список записей по выбранному делу с датой/временем и возможностью просмотра ответов каждой записи. Отображается на **странице просмотра дела** ([14-screen-deed-view.md](14-screen-deed-view.md)); при необходимости может быть вынесен в отдельный экран.

## Элементы экрана

- **Запись дела** — располагается в хронологии по времени, содержит дату, время и значение в превью-форме.
- **Навигация назад** — возврат к списку дел или к карточке данного дела.

## Сценарии

1. Пользователь открывает дело (страница просмотра дела) — видит блок истории записей по этому делу.
2. В списке видит все свои записи с датой, временем и превью.
3. Открывает любую запись и просматривает полные ответы и служебные поля (дата, время, заметки); при необходимости редактирует их.

## Вкладка «История» (маршрут `/history`)

Отдельный экран со списком записей по всем делам. Переход на просмотр записи (`/records/:id`) передаёт в location state признак `from: 'history'`, чтобы на экране записи показывать действие **переход к делу** (быстрый переход на страницу этого дела). Если запись открыта с **страницы дела** или иначе без этого признака, кнопка «к делу» в шапке не показывается — контекст уже привязан к делу или отличается.

На экране записи (`/records/:id`) в AppBar при просмотре заголовок «Запись», при редактировании — «Редактирование». В шапке: **Дублировать**, при необходимости кнопка **к делу** (с истории), меню **⋯** с пунктами «Добавить», «Редактировать», «Удалить».

В списках карточек записей (вкладка «История» и блок истории на странице дела) доступно **контекстное меню**: на тач-устройствах — **long tap** по карточке, в веб-браузере — **правая кнопка мыши** по карточке. Пункты: **Дублировать** (переход на форму заполнения с копией ответов), **Редактировать** (открытие записи сразу в режиме правки), **Удалить** (с подтверждением). Короткий тап / левый клик по-прежнему открывает просмотр записи.
